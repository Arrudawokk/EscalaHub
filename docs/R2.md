# Cloudflare R2

## Configuração

Crie um bucket privado no Cloudflare R2 e uma credencial S3 com permissão de leitura de objetos limitada a esse bucket.

Variáveis obrigatórias:

```env
R2_ACCOUNT_ID=
R2_ACCESS_KEY_ID=
R2_SECRET_ACCESS_KEY=
R2_BUCKET=
```

Variável opcional:

```env
R2_PUBLIC_URL=
```

`R2_PUBLIC_URL` não participa da entrega privada e deve permanecer vazia quando o bucket contém somente produtos vendidos.

## Onde obter

- `R2_ACCOUNT_ID`: página inicial da conta Cloudflare ou visão geral do R2.
- `R2_ACCESS_KEY_ID` e `R2_SECRET_ACCESS_KEY`: R2 → Manage R2 API Tokens → Create API token → credenciais S3.
- `R2_BUCKET`: nome exato do bucket privado criado no R2.

O Secret Access Key é exibido somente durante a criação. Armazene-o diretamente nas variáveis protegidas da Vercel e nunca no Git.

## Upload do e-book atual

No painel do bucket, crie a estrutura ou envie diretamente usando esta chave:

```text
products/trafego-pago-do-zero-a-escala/trafego-pago-do-zero-a-escala.pdf
```

O nome do arquivo local pode ser diferente; o que precisa coincidir com o catálogo é a chave final do objeto.

## Produção

1. Configure as quatro variáveis obrigatórias em Preview.
2. Faça upload do PDF na chave documentada.
3. Teste usuário sem compra: HTTP 403.
4. Teste chave inexistente: HTTP 404.
5. Teste pedido aprovado: redirect temporário para R2 e download como attachment.
6. Confirme que a URL inclui expiração de 300 segundos.
7. Repita em Production e faça uma compra controlada.

## Rotação

Crie uma nova credencial de leitura, atualize os ambientes, publique e valide um download. Somente depois revogue a credencial antiga. Substituir o PDF na mesma chave não exige deploy nem alteração de código.
